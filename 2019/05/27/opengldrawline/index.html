这里的问题有两个：

  • 效率低
  • t如何控制

t取大了画出来的并不是线，而是一堆点。t取小了会浪费，有很多重复的x和y。

我们想着要节约，就每次 x 增加1，然后来画y。

这样画线是对的因为我们假设 $y = mx + b $, 直线斜率m， 截距b

同时它的问题是我们也已经指出:

  • 如果直线斜率太大，比如 m = 3, 那么x每增加1个像素，y增加3个像素，这样画出来就是分离的点。
  • 只能适用于 x0 < x1的状况

尝试三

所以想法是：

  • 如有必要交换 x0 和 x1，这样使得 x0 一定小于 x1
  • 如果斜率比较大，则交换 x 和 y
